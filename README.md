## 📌 О проекте

Этот Telegram-бот был разработан для **Липецкой государственной филармонии** с целью автоматизации взаимодействия с аудиторией и упрощения управления контентом.

### Ключевые преимущества:

✅ **Автоматизация публикации афиш** — данные загружаются напрямую с официального сайта через JSON API  
✅ **Интерактивная система розыгрышей** — проведение конкурсов с автоматическим выбором победителей  
✅ **Поддержка пользователей** — сбор отзывов и оперативное реагирование на проблемы  
✅ **Гибкая админ-панель** — управление контентом без необходимости программирования  
✅ **Каталог исполнителей** — подробная информация о солистах, коллективах и концертмейстерах

## ✨ Возможности

### 🎫 Автоматическая публикация афиш

**Интеграция с сайтом филармонии:**
- Прямое получение данных через JSON API (`https://unionzal.ru/rest/export/json/afisha`)
- Парсинг HTML-описаний концертов с помощью BeautifulSoup
- Автоматическая фильтрация прошедших мероприятий
- Сортировка событий по дате проведения

**Категории событий:**
| Тип | Описание |
|-----|----------|
| 🎟️ **Билеты** | Разовые концерты с возможностью покупки билета |
| 📜 **Абонементы** | Циклы концертов (сезонные абонементы) |

**Функционал просмотра:**
- 🖼️ Показ баннера мероприятия
- 📅 Отображение даты и времени начала/окончания
- 🔗 Прямая ссылка на покупку билетов на Afisha.ru
- 📝 Подробное описание с авто-разбивкой на части (до 950 символов)
- ◀ ▶ Навигация между событиями
- 🔄 Автообновление кэша каждые 12 часов

---

### 🎁 Система розыгрышей

**Создание розыгрыша (администратор):**
1. Выбор формата:
   - Без изображений (только текст)
   - С 1 изображением (приз)
   - С 2 изображениями (приветствие + приз)
2. Ввод текста конкурса
3. Настройка параметров:
   - 📅 Дата окончания (календарь с выбором месяца/дня)
   - ⏰ Время объявления результатов
   - 🏆 Количество победителей (1–10)
4. Автоматическое или ручное завершение

**Участие пользователей:**
- ✅ Проверка подписки на канал `@filarmonia48`
- 🎯 Кнопка «Участвовать» в один клик
- 📬 Уведомление о успешном участии
- 🔔 Персональные сообщения о результатах

**Авто-результаты:**
- Автоматический выбор победителей в заданное время
- Использование `random.sample()` для честного розыгрыша
- Отправка результатов:
  - Победителям: персональное поздравление + фото приза
  - Остальным участникам: сообщение с поддержкой
  - Организатору: полный список победителей

**Администрирование активного розыгрыша:**
- 👥 Просмотр количества участников
- ⏹️ Завершение без победителя (технические причины)
- 🏆 Завершение с объявлением победителей вручную

## 🚀 Быстрый старт

### Требования

- **Python**: версия 3.10 или выше
- **Docker** и **Docker Compose** (опционально)
- **Токен Telegram-бота** (получить у [@BotFather](https://t.me/BotFather))
- **Доступ к интернету** (для загрузки данных с сайта филармонии)

### Шаг 1: Клонирование репозитория

```bash
git clone <repository-url>
cd <project-directory>
```

### Шаг 2: Установка зависимостей

#### Вариант A: Локальная установка

```bash
pip install -r requirements.txt
```

#### Вариант B: Использование Docker

```bash
docker-compose up --build
```

### Шаг 3: Настройка конфигурации

Откройте файл `settings.py` и укажите необходимые параметры:

```python
# Токен вашего Telegram-бота
TOKEN_BOT = "YOUR_BOT_TOKEN_HERE"

# URL для получения афиши (по умолчанию указан)
JSON_AFISHA_URL = "https://unionzal.ru/rest/export/json/afisha"

# URL для получения исполнителей (по умолчанию указан)
JSON_PERSONS_URL = "https://unionzal.ru/rest/export/json/persons"

# ID чата для уведомлений (опционально)
CHAT = "filarmonia48"
```

Создайте директорию и файлы для хранения данных:


### Хранилища данных (txt/)

| Файл | Описание | Формат |
|------|----------|--------|

| `giveway_fl.txt` | Данные текущего розыгрыша | Pickle (dict) |
| `admin_fl.txt` | Статусы администраторов | Pickle (list[dict]) |
| `wallpapers_fl.txt` | Изображения интерфейса | Pickle (dict) |

### Структура данных

**Пользователь (USERS):**
```python
[username, chat_id, full_name, user_id, user_html]
```

**Отзыв (FEEDBACK_USER):**
```python
{
    'id': int,
    'user_id': int,
    'full_name': str,
    'username': str,
    'user_html': str,
    'text': str,
    'read': bool
}
```

**Розыгрыш (START_GIVEAWAY):**
```python
{
    'active': bool,
    'contest_text': str,
    'picture': str,  # file_id
    'prize': str,    # file_id
    'author': dict,
    'participants': list[dict],
    'winners': int,
    'end_date': str,  # YYYY-MM-DD
    'end_time': str,  # HH:MM
    'auto': bool
}
```

## 🔒 Безопасность

### Аутентификация администраторов

- **Парольная защита**: Вход в админ-панель требует знания пароля
- **Сессионное хранение**: Статус администратора хранится в `context.user_data`
- **Проверка прав**: Функция `is_admin(user_id)` проверяет список администраторов
- **Выход из системы**: Кнопка «Выход» сбрасывает права до следующего входа

### Хранение данных

- **Pickle-сериализация**: Все данные сохраняются в бинарном формате
- **Локальное хранение**: Файлы находятся в директории `txt/`
- **Персистентность**: Данные сохраняются после перезапуска бота

### Рекомендации по безопасности

⚠️ **Важно:**
1. Не публикуйте файл `password_fl.txt` в открытом доступе
2. Используйте сложные пароли для админ-панели
3. Ограничьте доступ к серверу с ботом
4. Регулярно обновляйте зависимости (`pip install --upgrade -r requirements.txt`)
5. Добавьте `.env` файл для хранения токена (не коммитьте в Git!)
